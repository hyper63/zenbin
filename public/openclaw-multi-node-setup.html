🌐 OpenClaw Multi-Node Setup

Connect multiple devices to a single OpenClaw gateway

Documentation Guide

Architecture Overview

In OpenClaw, the Gateway is the brain that receives messages and runs the AI model. Nodes are peripheral devices that execute commands locally and provide capabilities like camera, screen recording, and shell access.

┌─────────────────────────────────────────────────────────────┐
│                        GATEWAY HOST                          │
│            (receives messages, runs model, routes)           │
│                                                              │
│   Telegram ──┐                                               │
│   WhatsApp ──┼──► Gateway WebSocket (port 18789)             │
│   Discord ───┘                                               │
│                                                              │
└───────────────────────────┬──────────────────────────────────┘
                            │
         ┌──────────────────┼──────────────────┐

   │  NODE 1  │      │  NODE 2  │      │  NODE 3  │
   │  (Mac)   │      │ (Linux)  │      │ (iPhone) │
   │          │      │          │      │          │
   │ • canvas │      │ • system │      │ • camera │

   │ • system │      │          │      │ • location│
   └──────────┘      └──────────┘      └──────────┘
Key Concept: The Gateway handles all AI logic and messaging. Nodes are "dumb" peripherals that execute commands sent by the gateway. This means one Gateway can orchestrate many Nodes simultaneously.

Setting Up a Headless Node Host

Headless nodes are perfect for Linux servers, CI/CD runners, or any machine without a GUI. They connect via WebSocket to your Gateway.

1

Get the Gateway Token

On your Gateway host, retrieve the authentication token:

On Gateway Host
# Find your gateway token
cat ~/.openclaw/openclaw.json | grep token

# Ensure gateway is accessible
openclaw gateway status
2

Install & Connect the Node

On each node machine, install OpenClaw and connect to your gateway:

On Node Host (Foreground - Testing)
# Export the gateway token
export OPENCLAW_GATEWAY_TOKEN="your-gateway-token-here"

# Connect to gateway
openclaw node run --host gateway-host.local --port 18789 --display-name "Build Server"
On Node Host (Service - Production)
export OPENCLAW_GATEWAY_TOKEN="your-gateway-token-here"

# Install as a system service
openclaw node install --host gateway-host.local --port 18789 --display-name "Build Server"

# Start the service
openclaw node start

# Manage the service
openclaw node status
openclaw node restart
openclaw node stop
3

Approve the Pairing Request

Back on the Gateway, approve the new node:

On Gateway Host
# List pending pairing requests
openclaw nodes pending

# Approve the new node
openclaw nodes approve <request-id>

# Verify all nodes are connected
openclaw nodes status
4

Configure Exec Allowlist

Each node needs its own allowlist for security:

On Gateway (manages all nodes)
# Allow specific commands on this node
openclaw approvals allowlist add --node "Build Server" "/usr/bin/git"
openclaw approvals allowlist add --node "Build Server" "/usr/bin/docker"
openclaw approvals allowlist add --node "Build Server" "/usr/bin/npm"

Remote Gateway via SSH Tunnel

If your Gateway binds to 127.0.0.1 (loopback only), remote nodes can't connect directly. Use an SSH tunnel:

On Node Host
# Terminal A: Create SSH tunnel to gateway
ssh -N -L 18790:127.0.0.1:18789 user@gateway-host

# Terminal B: Connect through the tunnel
export OPENCLAW_GATEWAY_TOKEN="your-token"
openclaw node run --host 127.0.0.1 --port 18790 --display-name "Remote Node"
Security Note: The gateway token is a secret. Never commit it to version control or share it publicly.

Binding Exec to Specific Nodes

Configure which node handles exec commands by default:

Global Default
openclaw config set tools.exec.host node
openclaw config set tools.exec.security allowlist
openclaw config set tools.exec.node "Build Server"
Per-Agent Override
openclaw config get agents.list
openclaw config set agents.list[0].tools.exec.node "Build Server"
Runtime Override (in chat)
/exec host=node node="Build Server"

Workflow

On CI Builder (Linux)
export OPENCLAW_GATEWAY_TOKEN="abc123..."
openclaw node run --host macmini.lan --display-name "CI Builder"
On Gateway (approve nodes)
openclaw nodes pending
openclaw nodes approve <ci-builder-request-id>

# List all connected nodes
openclaw nodes status

# OUTPUT:
#   Thomas's Mac mini (macOS) - paired
#   CI Builder (Linux) - paired
#   Thomas's iPhone (iOS) - paired

Using Nodes

Run Commands on Specific Node
# Run docker command on CI Builder
openclaw nodes run --node "CI Builder" -- docker ps

# Take photo from iPhone
openclaw nodes camera snap --node "Thomas's iPhone" --facing back

# Get screen recording from Mac
openclaw nodes screen record --node "Thomas's Mac mini" --duration 10s

Key Files

~/.openclaw/node.json Node identity, token, and connection info
~/.openclaw/exec-approvals.json Allowed commands for this node
~/.openclaw/nodes/paired.json Gateway's registry of paired nodes
~/.openclaw/openclaw.json Gateway config (contains auth token)

Security Best Practices

🔑 Token Security

Treat all tokens as secrets. Never commit to VCS or share publicly.

📋 Per-Node Allowlists

Each node has its own allowlist. Scope commands to minimum required.

📱 Foreground Required

iOS/Android apps must be in foreground for camera and canvas operations.

🔄 Token Rotation

Re-pair a node to rotate its token. Old tokens are invalidated.

CLI Reference

Node Management Commands
# Check node status
openclaw nodes status
openclaw nodes describe --node "Build Server"

# Rename a node
openclaw nodes rename --node <id|ip> --name "New Name"

# Node host service
openclaw node run --host gateway.host --port 18789
openclaw node install --host gateway.host --display-name "CI"
openclaw node start # start installed service
openclaw node stop # stop service
openclaw node restart # restart service
openclaw node uninstall # remove service
Node Capabilities
# Remote commands
openclaw nodes run --node "Server" -- uname -a
openclaw nodes notify --node "Server" --title "Alert" --body "Job done"

# Camera
openclaw nodes camera snap --node "iPhone" --facing back
openclaw nodes camera clip --node "iPhone" --duration 10s

# Screen recording
openclaw nodes screen record --node "Mac" --duration 30s --fps 10

# Canvas (browser control)
openclaw nodes canvas present --node "iPad" --target https://example.com
openclaw nodes canvas snapshot --node "iPad" --format png

# Location
openclaw nodes location get --node "iPhone" --accuracy precise
